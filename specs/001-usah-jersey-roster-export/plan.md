# Implementation Plan: USA Hockey Readiness — Jersey Numbers, Association IDs & CSV Roster Export

**Branch**: `001-usah-jersey-roster-export` | **Date**: 2026-03-19 | **Spec**: [spec.md](./spec.md)

---

## Summary

Add jersey numbers and USA Hockey Member IDs to player and team official records, then expose a one-click CSV roster export for team admins. This involves two nullable DB columns on `Player` (jersey number + USAH ID) and one on `TeamMember` (USAH ID), extensions to existing Zod validation schemas and Server Actions, a new Route Handler for CSV download, and targeted UI updates to the existing `AddPlayerDialog`, `PlayerCard`, and `RosterList` components.

---

## Implementation Phases

### Phase 1 — Data Layer (DB + Validation + Actions)

**Tasks**:

1. **Prisma schema migration**
   - Add `jerseyNumber Int?` to `Player`
   - Add `usahMemberId String? @db.VarChar(20)` to `Player`
   - Add `usahMemberId String? @db.VarChar(20)` to `TeamMember`
   - Run `bun run db:migrate` (name: `add_jersey_number_usah_member_id`)
   - Run `bun run db:generate`

2. **Extend Zod schemas** (`lib/utils/validation.ts`)
   - Add `jerseyNumber` and `usahMemberId` to `addPlayerSchema`
   - `updatePlayerSchema` inherits via `.extend()` — no change needed
   - Add `updateTeamMemberUsahIdSchema`
   - Export new types: `UpdateTeamMemberUsahIdInput`

3. **Update `addPlayer` action** (`lib/actions/roster.ts`)
   - Pass `jerseyNumber` and `usahMemberId` in `prisma.player.create()`
   - After create, query team for duplicate jersey number (same teamId, same number, different player id)
   - Return `warning` string in success response if duplicate found

4. **Update `updatePlayer` action** (`lib/actions/roster.ts`)
   - Pass `jerseyNumber` and `usahMemberId` in `prisma.player.update()`
   - Duplicate jersey check (exclude the current player's id from the query)
   - Return `warning` string in success response if duplicate found

5. **Add `updateTeamMemberUsahId` action** (`lib/actions/roster.ts`)
   - Validate input with `updateTeamMemberUsahIdSchema`
   - `requireTeamAdmin(teamId)`
   - Verify `TeamMember` belongs to the given team
   - `prisma.teamMember.update({ where: { id }, data: { usahMemberId } })`
   - `revalidatePath('/roster')`

6. **Update `Player` type** (`types/roster.ts`)
   - Add `jerseyNumber: number | null` to base `Player` type
   - Add `usahMemberId: string | null` to base `Player` type (document as admin-only)
   - Add `TeamMemberWithUser` type

---

### Phase 2 — CSV Export Route Handler

**Tasks**:

1. **Create CSV utility** (inline in route or extracted to `lib/utils/csv.ts`)
   - `escapeCsvField(value: string): string` — wraps in quotes if contains comma/quote/newline, escapes embedded quotes
   - `toCsvRow(fields: (string | null | number)[]): string` — joins escaped fields with commas
   - `toCsvContent(headers: string[], rows: (string | null | number)[][]): string` — builds full CSV string with UTF-8 BOM

2. **Create Route Handler** (`app/api/roster/export/route.ts`)
   - `GET` handler — reads `teamId` from `searchParams`
   - Auth: call `requireTeamAdmin(teamId)` — returns 403 on failure
   - Query players: `prisma.player.findMany({ where: { teamId }, select: { id, name, email, jerseyNumber, usahMemberId }, orderBy: [...] })`
   - Query team officials: `prisma.teamMember.findMany({ where: { teamId, role: 'ADMIN' }, select: { usahMemberId, user: { select: { name, email } } } })`
   - Build CSV rows (officials first, then players sorted by jerseyNumber asc nulls last, then name)
   - Return `new Response(csvContent, { headers: { 'Content-Type': 'text/csv; charset=utf-8', 'Content-Disposition': 'attachment; filename="..."' } })`

### Phase 3 — UI Updates

**Tasks**:

1. **`AddPlayerDialog.tsx`** — add two new fields
   - Jersey number: `TextField` with `type="number"`, `inputProps={{ min: 1, max: 99 }}`
   - USAH Member ID: `TextField` (text input, shown to admin only — `AddPlayerDialog` is already admin-only)
   - Pre-populate both fields from `player` prop when editing
   - Show `warning` toast if success response includes `warning`

2. **`PlayerCard.tsx`** — display jersey number and (for admins) USAH ID
    - Render `#{jerseyNumber}` badge/chip next to player name if set
    - In admin view, show USAH Member ID field with lock icon (consistent with emergency contact display)

3. **`RosterList.tsx`** — add Export CSV button
    - Admin-only `Button` with download icon
    - On click: `window.location.href = /api/roster/export?teamId=${teamId}` (triggers browser download)
    - Or `<a href="...">` — simpler, no JS needed

4. **Update `RosterList` data fetch** (in `app/(dashboard)/roster/page.tsx` or wherever the query lives)
    - Include `jerseyNumber` in the non-admin player select projection
    - Include `jerseyNumber` and `usahMemberId` in the admin player select projection

### Phase 4 — Tests

**Tasks**:

1. **`validation.test.ts`** — new test cases
    - Jersey number: valid (1, 50, 99), invalid (0, 100, -1, non-integer)
    - USAH Member ID: valid (alphanumeric, trimmed whitespace), invalid (>20 chars, special chars)

2. **`roster.test.ts`** — new test cases
    - `addPlayer` with jersey number → saved correctly
    - `addPlayer` with duplicate jersey number → success + `warning` in response
    - `updatePlayer` clearing jersey number (null) → saved as null
    - `updateTeamMemberUsahId` → happy path + unauthorized

3. **`export.test.ts`** — new test file
    - Unauthenticated request → 401
    - Non-admin request → 403
    - Missing teamId → 400
    - Empty roster → CSV with headers only
    - Full roster → correct columns, correct row order, RFC 4180 encoding
    - Name with comma → field is quoted in CSV output

## Risk Notes

| Risk | Mitigation |
| --- | --- |
| `requireTeamAdmin` throws (redirects) in Server Actions but Route Handlers need a different error response | Route Handler must catch the redirect/throw from auth and return HTTP 403 explicitly |
| `window.location.href` approach for CSV download won't work if the session cookie is not sent | Use `<a href>` tag — browser sends cookies automatically for same-origin requests |
| MUI TextField `type="number"` allows non-integer floats on some browsers | Add Zod `.int()` validation on server; add `step="1"` to input props |
